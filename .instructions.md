# FireStop Evacuation Tracker - Workspace Instructions

## Project Overview

FireStop Evac Tracker is an ASP.NET Core 8 Razor Pages web application for tracking evacuation diagram jobs. It manages job creation, PDF uploads, status tracking, client approvals, and job notes with built-in authentication.

**Key Features:**
- Create and manage evacuation jobs
- Auto-generate unique job names
- Upload/manage draft PDFs
- Track job status (New, In Progress, Completed, Approved)
- Client approval workflow with share codes
- Job notes with user tracking
- PostgreSQL/SQLite database support
- Docker containerization support

## Tech Stack

- **Framework:** ASP.NET Core 8 (.NET 8)
- **UI:** Razor Pages (server-side rendering)
- **ORM:** Entity Framework Core 8.0.8
- **Database:** SQLite (development) / PostgreSQL (production)
- **Authentication:** Session-based
- **Containerization:** Docker & Docker Compose
- **Styling:** Bootstrap 5 (inferred from layout pattern)

## Development Setup

### Prerequisites
- Visual Studio 2022 or VS Code with C# support
- .NET 8 SDK
- PostgreSQL (for production) or SQLite (default for development)
- Docker & Docker Compose (optional, for containerized development)

### Initial Setup

1. **Clone/Open Project**
   ```bash
   # If using VS Code
   code FireStopEvacTracker.csproj
   # OR open in Visual Studio
   ```

2. **Configure Database Connection**
   - Edit `appsettings.json`
   - Default uses SQLite (good for development)
   - To use PostgreSQL, update the connection string:
     ```json
     "DefaultConnection": "Host=localhost;Port=5432;Database=firestop_evac_tracker;Username=postgres;Password=YOUR_PASSWORD"
     ```

3. **Restore Dependencies**
   ```bash
   dotnet restore
   ```

4. **Apply Database Migrations**
   ```bash
   dotnet ef database update
   ```
   Or in Visual Studio Package Manager Console:
   ```powershell
   Update-Database
   ```

5. **Run Application**
   ```bash
   dotnet run
   # App runs at https://localhost:5001
   ```

6. **Login with Demo Account**
   - Username: `admin`
   - Password: `admin123`

## Folder & File Guidelines

### Adding New Pages
When adding a new Razor Page:
1. Create `.cshtml` file in `/Pages` with folder structure matching the URL route
2. Create corresponding `.cshtml.cs` PageModel class
3. Add navigation links in `_Layout.cshtml`

**Example:** For route `/reports/monthly`, create:
- `Pages/Reports/Monthly.cshtml`
- `Pages/Reports/Monthly.cshtml.cs`

### Database Models
- Keep models in `/Models/` folder
- Use `DataAnnotations` for validation
- Use navigation properties for relationships
- ModelSnapshot auto-generated in `/Migrations/`

### Services
- Keep business logic in `/Services/`
- Register in `Program.cs` with `AddScoped<>` or `AddTransient<>`
- Use dependency injection in Pages via `@inject` or constructor

### Static Files
- CSS in `/wwwroot/css/`
- Images in `/wwwroot/img/`
- **User uploads go to `/wwwroot/uploads/`** (PDF files)

## Common Development Tasks

### Create New Database Migration
```bash
dotnet ef migrations add YourMigrationName
dotnet ef database update
```

### Add New Service
1. Create class in `Services/` folder
2. Implement service methods
3. Register in `Program.cs`:
   ```csharp
   builder.Services.AddScoped<YourService>();
   ```
4. Inject in Pages using `@inject` or constructor DI

### Add New Razor Page with Form
1. Create `/Pages/YourFeature/YourPage.cshtml` and `.cshtml.cs`
2. Define model binding in PageModel:
   ```csharp
   [BindProperty]
   public YourModel Model { get; set; }
   ```
3. Create form in `.cshtml` with `asp-for` tag helpers
4. Handle `OnPostAsync()` in PageModel

### Upload PDF File
- Use `PdfStorageService.SavePdfAsync()` method
- Files stored in `/wwwroot/uploads/[JobName]/`
- Path tracked in `EvacJob.DraftPdfPath` property

### Check User Authentication
- User ID stored in session: `HttpContext.Session.GetInt32("UserId")`
- Use `AuthService` for login/logout logic
- Unauthenticated users redirect to `/Login`

## Coding Conventions

### Naming
- **Classes:** PascalCase (`EvacJob`, `JobNameService`)
- **Properties:** PascalCase (`ClientName`, `DraftPdfFileName`)
- **Methods:** PascalCase (`GetShortJobName()`)
- **Private fields:** camelCase or `_camelCase`
- **Pages:** PascalCase with folder structure matching route

### C# Style
- Use nullable reference types (enabled in `.csproj`)
- Use implicit usings (enabled in `.csproj`)
- Use async/await for I/O operations (`async Task`, `await`)
- Use LINQ where practical
- Add XML documentation for public methods

### Database
- Use migrations for schema changes (never manual SQL in production)
- Add indexes for frequently-searched fields
- Use relationships instead of raw IDs
- Cascade delete for dependent entities

### Razor Pages
- Keep views focused; move logic to PageModel
- Use tag helpers: `asp-for`, `asp-action`, `asp-controller`
- Use `@inject` for service dependencies
- Use `@model` to reference PageModel class

## Database Schema

### Core Entities

**EvacJob** - Main job entity
- `Id` (Primary Key)
- `ClientName`, `SiteAddress`, `JobName`
- `Status` (enum: New, InProgress, Completed, Approved)
- `DraftPdfPath`, `DraftPdfFileName`
- `ShareCode` (for client approval link)
- `CreatedAt`, `UpdatedAt`, `StatusUpdatedAt`
- Relations: JobNotes, Approvals (1:Many)

**JobNote** - Comments/notes on jobs
- `Id` (Primary Key)
- `EvacJobId` (Foreign Key)
- `Content`, `AddedBy`
- `CreatedAt`

**JobApproval** - Client approval records
- `Id` (Primary Key)
- `JobId` (Foreign Key)
- `ClientEmail`, `ClientName`
- `ApprovedAt`

**User** - System users
- `Id` (Primary Key)
- `Username`, `PasswordHash`
- `CreatedAt`

### Migrations
- Located in `/Migrations/`
- Auto-generated by `dotnet ef migrations add`
- Apply with `dotnet ef database update`
- Use descriptive names: `AddStatusUpdatedAt`, `AddJobApproval`

## Troubleshooting

**Migration Issues:**
- If migration fails, check `AppDbContext.OnModelCreating()` for schema constraints
- Verify connection string is correct
- Clear `bin/` and `obj/` folders, then rebuild

**PDF Upload Issues:**
- Verify `/wwwroot/uploads/` folder exists and is writable
- Check `PdfStorageService` implementation for path creation
- Review `DraftPdfPath` and `DraftPdfFileName` in EvacJob model

**Authentication Problems:**
- Session data stored locally (no distributed cache configured)
- Check `_Layout.cshtml` for login redirect
- Verify `AuthService` password validation logic

**Database Connection:**
- SQLite: Ensure file permissions in project directory
- PostgreSQL: Verify server is running and credentials are correct
- Use `dotnet ef database update` to diagnose migration issues
